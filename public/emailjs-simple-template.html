EmailJS Simple Template Setup Guide

This document provides instructions for setting up the EmailJS template to work with our direct HTML email approach.

Important: This approach sends a complete HTML email directly from our code, bypassing most of EmailJS's template system. We only need a minimal template in EmailJS that passes our HTML content through.

EmailJS Template Setup Steps

  1. Log in to EmailJS dashboard at https://dashboard.emailjs.com/
  2. Go to Email Templates and either create a new template or edit your existing one
  3. Set a template name that matches what you've configured in your environment variables (NEXT_PUBLIC_EMAILJS_TEMPLATE_ID)
  4. Use this minimal template code: 
    <!-- This minimal template simply passes through our HTML email content -->
{{content}}
  5. Alternative (if the minimal template doesn't work): 
    <!-- Fallback template with basic structure and plaintext fallback -->
<h1>{{subject}}</h1>

{{content}}

<hr>

<h3>Plaintext Version:</h3>
<pre>{{message}}</pre>
  6. Save the template and make sure the template ID matches your NEXT_PUBLIC_EMAILJS_TEMPLATE_ID environment variable

How It Works

Our application now generates a complete HTML email in code and passes it to EmailJS in the content variable. The minimal template above just outputs that HTML content directly. If EmailJS doesn't handle the HTML properly, the alternative template provides a basic structure with a plaintext fallback.

Troubleshooting

  • If emails are still not formatted properly: Check if your email client supports HTML emails and that EmailJS is not stripping HTML content.
  • If you see HTML code in your emails: Your email client might be displaying the raw HTML instead of rendering it. Try the alternative template approach.
  • If some email clients show HTML and others don't: This is normal, as not all email clients support the same HTML features. Our approach includes a plaintext fallback for this reason.
